## Tab Navigation Fix

`tabBarTestID` props were already set in `app/(tabs)/_layout.tsx`, but Expo Go does not surface them as `resource-id` in the iOS accessibility hierarchy. Maestro's `tapOn: { id: "..." }` therefore failed.

**Fix applied:** Switched YAML selectors from `id: "tab-discover"` to accessibility-label matching: `tapOn: "Discover, tab, 2 of 6"` (the string iOS generates for tab bar items). All five tabs now navigate correctly.

## Visual Issues Found

### P1 — Brand name truncation on Discover cards

- **Where:** `04-discover__initial.png` — Curated Edits rail
- **What:** "REGIME DES FLEU..." — brand "Regime des Fleurs" is truncated with ellipsis on the card.
- **Note:** The commit `4bdf74a` reduced letter-spacing for brands >15 chars. "REGIME DES FLEURS" (17 chars uppercase-spaced) still overflows. The fix works for "OLFACTIVE STUDIO" (seen fitting correctly on `03-home__bottom.png`) but not for this brand.
- **Suggested fix:** Either increase the max-width for brand text, reduce font size for brands >16 chars, or allow 2-line brand wrapping.

### P2 — Celebrity subtitle truncation

- **Where:** `04-discover__initial.png` — Celebrity Picks rail (visible in earlier run screenshot, not captured in this run because the scroll missed it)
- **What:** "Marilyn Monroe, Nico..." — the comma-joined celebrity names overflow the single line.
- **Suggested fix:** Allow `numberOfLines={2}` on the subtitle, or cap to first 2 celebrity names + "& more".

### P2 — Fragrance name includes concentration suffix

- **Where:** `02-home__after-scroll.png` — New Arrivals rail
- **What:** Card shows "Y2K(R) - Extrait de Parfum" instead of just "Y2K(R)". The `cardDisplayName()` pipe-stripping logic splits on `|` but this name uses ` - ` as a separator.
- **Suggested fix:** Extend `cardDisplayName()` to also strip ` - Extrait de ...` / ` - Eau de ...` suffixes.

### P2 — "ROOM NO." appears truncated or incomplete

- **Where:** `02-home__after-scroll.png`, `03-home__bottom.png` — Today's Edit rail
- **What:** Card shows "ROOM NO." which looks like it should be "ROOM NO. 1015" or similar. May be the actual name in the database, or may be truncated.
- **Action:** Verify in Supabase whether the full name is just "Room No." or if truncation is occurring.

### P3 — Splash/launch overlay captured in screenshot 01

- **Where:** `01-home__initial.png`
- **What:** The Expo Go app icon overlay is still visible (launch animation not fully dismissed before screenshot). The "Error configuring Purchases: Error: Invalid A..." RevenueCat banner is also visible at the bottom.
- **Note:** RevenueCat error is expected (no StoreKit config). The splash overlay is a timing issue in the test — the `extendedWaitUntil` for `WEAR TODAY` passed, but the launch animation was still animating out.
- **Suggested fix (test only):** Add a short `- waitForAnimationToEnd` or increase the initial wait before the first screenshot.

### P3 — Profile monogram clipped

- **Where:** `09-profile__top.png`
- **What:** The cursive monogram letter inside the profile avatar circle appears slightly clipped at the bottom edge. Minor but noticeable.

### Info — "DEV TOOLS" section visible in Profile
